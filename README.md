# Code and data needed for reproduction

This folder contains the code used to construct the HTS-derived AI evaluation dataset and to run the main benchmark analyses. The repository is intended as proof of code and data processing for the NeurIPS dataset submission. Large raw virtual-screening libraries are not suitable for direct GitHub storage; they should be referenced through their public/vendor sources or deposited separately when redistribution is allowed.

## Required input files and order

To reproduce the full workflow, the following inputs are needed and should be run in this order.

### 1. Initial virtual-screening library clustering

For each VS library listed in `1.Libraries-initial-clustering/README.md`, provide one of the following input formats in that library's working directory.

If input is in SMILES (`*.smi`) format, run:

- `1.Libraries-initial-clustering/2-clustering.py`

If input is in SDF (`*.sdf`) format, run:

- `1.Libraries-initial-clustering/1-sdf-to-npy.py` to generate packed fingerprint chunks (`*.npy`)
- `1.Libraries-initial-clustering/2-clustering-sdf.py` to cluster the fingerprint chunks

The output from this stage is one medoid file per library:

```text
Libraries/<library_name>/npy_medoids.pkl
```

Each `npy_medoids.pkl` must contain:

- `fingerprints`: unpacked 2048-bit ECFP4/Morgan radius 2 medoid fingerprints.
- `medoid_indices`: original row indices of selected medoids.
- `cluster_size`: number of original fingerprints represented by each medoid.

For UF-Scripps actives and decoys already available as tagged CSV files, use `1.Libraries-initial-clustering/make_pkl.py` to write a BitBirch-compatible `npy_medoids.pkl` where each compound is treated as a medoid with `cluster_size = 1`. The script expects columns named `Title` and `SMILES`.

### 2. Active analog and class-separation analysis

The scripts in `2.Section-3.0-Active-Analogs/` quantify active/decoy chemical similarity and help document the non-trivial separation problem. This step expects:

- active and decoy CSV files with `Title`, `SMILES`, and `Label` columns.
- ECFP4 fingerprints generated from SMILES.
- BitBirch-compatible active and decoy PKLs when clustering or medoid-level summaries are used.

Main files:

- `actives_prop_decoys.py`: compares active and decoy chemical space, computes nearest-neighbor/cluster composition summaries, and produces active-decoy overlap plots.
- `actives_left_outs_analysis.ipynb`: notebook used to inspect active analogs and compounds left out by the selected dataset construction strategy.

### 3. VS-space sampling approaches

The scripts in `3.Section-3.1-Approaches-VS-space/` expect:

- `./Libraries/<library_name>/npy_medoids.pkl` for all VS libraries.
- `./Libraries/UF-Scripps-Actives/npy_medoids.pkl` for active compounds when used by the approach scripts.
- `./Libraries/UF-Scripps-Decoys/npy_medoids.pkl` for decoy compounds when used by the approach scripts.

Use `3.Section-3.1-Approaches-VS-space/Read_me.md` for the exact commands. In short:

1. `re-clustering.py` implements the simple re-clustering approach.
2. `umap_libs.py` builds UMAP embeddings of medoid space.
3. `ratio_1_cap_study.py` samples decoys from UMAP space under cap settings.
4. `4-KMeans.py` runs the MiniBatchKMeans-based alternative.
5. `score.py`, `mean_nn.py`, `distros_phys.py`, and `plot_cached_descriptor_distributions.py` score the selected decoy sets.

## Scoring

Candidate decoy CSVs generated by `ratio_1_cap_study.py` are scored and compared against the original VS medoid space. The selected final construction uses:

- Descriptor cache `all_library_descriptor_cache.npz` when using `plot_cached_descriptor_distributions.py`.
- `mean_nn.py` to compare the selected set against medoids from each VS library.
- `score.py` to summarize cap-study metrics.This will generate a Heatmap from where the best combination of Cap max and Cap med can be selected. 
- `distros_phys.py` and `plot_cached_descriptor_distributions.py` to compare physicochemical descriptor distributions.

### 5. Model evaluation

The scripts in `4.Deep-Learning/` expect train/validation/test split CSVs with at least:

- `Title`
- `SMILES`
- `Label`
- `split` with values `train`, `val`, and `test`

The random forest script uses ECFP4 fingerprints. The GNN script expects the local `BMPNNs` package/module to be importable. See `4.Deep-Learning/README.md`.

The code uses Python with:

- `numpy`
- `pandas`
- `rdkit`
- `bblean` source in: https://github.com/mqcomplab/bblean
- `iSIM` source in: https://github.com/mqcomplab/iSIM
- `scikit-learn`
- `scipy`
- `matplotlib`
- `seaborn`
- `plotly`
- `matplotlib-venn`
- `torch` for GNN evaluation
- `Pytorch-Geometric` for GNN evaluation

## Suggested reproduction order

1. Cluster each raw VS library with the scripts in `1.Libraries-initial-clustering/` to generate `npy_medoids.pkl` files.
2. Run the active analog/class-separation analysis in `2.Section-3.0-Active-Analogs/`.
3. Run the Section 3.1 sampling approaches in `3.Section-3.1-Approaches-VS-space/Read_me.md`.
4. Score candidate decoy sets with `score.py`, `mean_nn.py`, and descriptor-distribution scripts.
5. Run model evaluation scripts in `4.Deep-Learning/` using the final split columns.
